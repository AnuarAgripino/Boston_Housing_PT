Questão 1 - Observação de Atributos¶

Um aumento no valor do RM pode sim influênciar no valor do MEDV, uma vez que, na maioria dos casos uma casa que possui mais quartos tende a ter um valor maior, e dessa forma, também há aqueles dispostos a pagar mais por ter mais quartos. Portanto, o aumendo do RM pode sim aumentar o valor do MEDV.

Provável que haja uma diminuição no valor da casa, visto que, em uma vizinhança de "classe baixa" o valor dos imóveis tem de ser acessiveis a renda dos trabalhadores. Dessa forma, um aumento do LSTAT pode sim diminuir o valor do MEDV.

Em resumo, para todos os casos citados, uma variação, seja ela de perca ou aumento é previsto que haja um efeito oposto ao valor do imóvel.

Implementação: Definir uma Métrica de Desempenho¶

É difícil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso é geralmente feito utilizando algum tipo de métrica de desempenho, através do cálculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida útil. Para este projeto, você irá calcular o coeficiente de determinação, R2, para quantificar o desempenho do seu modelo. O coeficiente de determinação é uma estatística útil no campo de análise de regressão uma vez que descreve o quão "bom" é a capacidade do modelo em fazer estimativas.

Os valores para R2 têm um alcance de 0 a 1, que captura a porcentagem da correlação ao quadrado entre a estimativa e o valor atual da variável alvo. Um modelo R2 de valor 0 sempre falha ao estimar a variável alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a variável alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da variável alvo (ao utilizar o modelo) que pode ser explicada pelos atributos. Um modelo pode dar também um R2 negativo, que indica que o modelo não é melhor do que aquele que estima ingenuamente a média da variável alvo.

Questão 5 - Equilíbrio entre viés e variância¶

  • Quando o modelo é treinado com o profundidade máxima 1, será que o modelo sofre mais de viés (erro sistemático) ou variância (erro aleatório)?
  • E o que acontece quando o modelo é treinado com profundidade máxima 10?

Dica: Como você sabe que um modelo está experimentando viés alto ou variância alta? Viés alto é um sinal de underfitting (o modelo não é complexo o suficiente para aprender os dados) e alta variância é um sinal de overfitting (o modelo está "decorando" os dados e não consegue generalizar bem o problema). Pense em modelos (com profundidade de 1 e 10, por exemplo) e qual deles está alinhado com qual parte do equilíbrio.

Resposta:

O modelo que inicialmente treinado com uma profundidade máxima de 1, sofre alto viés (underfitting) devido a baixa complexidade para filtrar o conjunto, ou seja, temos uma solução simples para um conjunto complexo. Quando treinado com profundidade máxima de 10, ocorre uma alta variância (overfitting) quando o algoritmo possui uma alta complexidade, dessa forma, temos uma solução muito complexa e sensível para um conjunto simples.

Tais fatores podem ser observado, por exemplo, quando a profundidade máxima é 1 o modelo possui um alto viés, baixo Score, ou seja o modelo não possui um ajuste adequado aos dados, sendo assim muito simples. De outro modo, quando possuimos uma profundidade máxima de 10, ocorre uma alta variância, ou seja, o modelo é sensível e complexo demais para os dados, vemos que o Training Score e o Validation Score possuem uma distância significativa entre ambos. Além do mais, ao observar veremos que os dados de treinamento chegaram mais perto do Score 1 enquanto os de Validação se distânciaram.

Portando, o ideal é que encontremos uma profundidade que alcance um Score próximo a 1, além do fato de que a pontuação de treino e validação também estejam bem próximas. Dessa forma, teremos um modelo que se ajusta corretamente aos dados. Sendo assim, para este modelo o ideal seria uma profundidade de 3, pois ele chega a um Score de aproximadamente 75%.

Questão 8 - Validação Cruzada¶

  • O que é a técnica de treinamento de validação-cruzada k-fold?
  • Quais benefícios essa técnica proporciona para busca em matriz ao otimizar um modelo?

Dica: Lembre-se de expllicar o que significa o 'k' da validação-cruzada k-fold, como a base de dados é dividida e quantas vezes ela é executada.

Assim como há um raciocínio por trás de utilizar um conjunto de teste, o que poderia dar errado ao utilizar busca em matriz sem um conjunto de validação cruzada? Você pode utilizar a documentação para basear sua resposta.

Resposta:

A técnica de validação-cruzada k-fold consiste nos seguintes passos:

  • 1- Dividir o conjunto total de dados em um número 'k' de subconjuntos;
  • 2- Um dos subconjuntos é usado para teste, enquanto os outros subconjuntos restantes (k - 1) são utilizados para treinamento;
  • 3- Esse processo se repete 'k' vezes até que todos os conjuntos tenham passado por este processo, e dessa forma, seja possível calcular a precisão do modelo;
  • 4- Em seguida, calculamos a média dos resultados para obter um resultado final.

Ou seja, durante todo esse processamento estamos utilizando o conjunto inteiro de dados, sendo dividido em 'k' vezes e realizando validação e teste. Dessa forma, utilizamos cada porção dos dados, ao invés de um único subconjunto específico, sem deixar de lado qualquer dado útil para o treinamento.

Apesar do custo de processamento computacional ser caro, acaba por ser viável, pois em projetos onde nos deparamos com uma quantidade de dados pequena podemos validar e testar de forma precisa com a quantidade limitada de dados. Entretanto, é recomendável que o número 'k' de divisões não seja alto, uma vez que sempre voltamos a estaca zero quando testamos/validamos um novo subconjunto (k - 1), o que torna o custo computacional caro. Além do mais, um número alto de divisões também pode acabar por diminuir a precisão.

A busca em matriz (GridSearch) faz uso dos conjuntos de dados de validação para comparar qual o melhor modelo, por exemplo, o F1 Score que é calculado no conjunto de validação cruzada em cada um dos modelos, sem ele não há como identificar qual modelo possui um melhor ajuste aos dados.

from sklearn.metrics import make_scorer
from sklearn.tree import DecisionTreeRegressor
from sklearn.model_selection import GridSearchCV

def fit_model(X, y):
    """ Desempenhar busca em matriz sobre o parâmetro the 'max_depth' para uma 
        árvore de decisão de regressão treinada nos dados de entrada [X, y]. """
    
    # Gerar conjuntos de validação-cruzada para o treinamento de dados 
    cv_sets = ShuffleSplit(n_splits = 10, test_size = 0.20, random_state = 0)

    # TODO: Gerar uma árvore de decisão de regressão de objeto
    regressor = DecisionTreeRegressor()

    # TODO: Gerar um dicionário para o parâmetro 'max_depth' com um alcance de 1 a 10
    params = {'max_depth': range(1, 11)}

    # TODO: Transformar 'performance_metric' em uma função de pontuação utilizando 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Gerar o objeto de busca em matriz
    grid = GridSearchCV(estimator=regressor, param_grid=params, scoring=scoring_fnc, cv=cv_sets)

    # Ajustar o objeto de busca em matriz com os dados para calcular o modelo ótimo
    grid = grid.fit(X, y)

    # Devolver o modelo ótimo depois de realizar o ajuste dos dados
    return grid.best_estimator_
