difcil medir a qualidade de um modelo dado sem quantificar seu desempenho durante o treinamento e teste. Isso geralmente feito utilizando algum tipo de mtrica de desempenho, atravs do clculo de algum tipo de erro, qualidade de ajuste, ou qualquer outra medida til. Para este projeto, voc ir calcular o coeficiente de determinao, R2, para quantificar o desempenho do seu modelo. O coeficiente de determinao uma estatstica til no campo de anlise de regresso uma vez que descreve o quo "bom" a capacidade do modelo em fazer estimativas.

Os valores para R2 tm um alcance de 0 a 1, que captura a porcentagem da correlao ao quadrado entre a estimativa e o valor atual da varivel alvo. Um modelo R2 de valor 0 sempre falha ao estimar a varivel alvo, enquanto que um modelo R2 de valor 1, estima perfeitamente a varivel alvo. Qualquer valor entre 0 e 1 indica qual a porcentagem da varivel alvo (ao utilizar o modelo) que pode ser explicada pelos atributos. Um modelo pode dar tambm um R2 negativo, que indica que o modelo no melhor do que aquele que estima ingenuamente a mdia da varivel alvo.

Dica: Como voc sabe que um modelo est experimentando vis alto ou varincia alta? Resposta:

O modelo que inicialmente treinado com uma profundidade mxima de 1, sofre alto vis (underfitting) devido a baixa complexidade para filtrar o conjunto, ou seja, temos uma soluo simples para um conjunto complexo. Quando treinado com profundidade mxima de 10, ocorre uma alta varincia (overfitting) quando o algoritmo possui uma alta complexidade, dessa forma, temos uma soluo muito complexa e sensvel para um conjunto simples.

Tais fatores podem ser observado, por exemplo, quando a profundidade mxima 1 o modelo possui um alto vis, baixo Score, ou seja o modelo no possui um ajuste adequado aos dados, sendo assim muito simples. De outro modo, quando possuimos uma profundidade mxima de 10, ocorre uma alta varincia, ou seja, o modelo sensvel e complexo demais para os dados, vemos que o Training Score e o Validation Score possuem uma distncia significativa entre ambos. Alm do mais, ao observar veremos que os dados de treinamento chegaram mais perto do Score 1 enquanto os de Validao se distnciaram.

Portando, o ideal que encontremos uma profundidade que alcance um Score prximo a 1, alm do fato de que a pontuao de treino e validao tambm estejam bem prximas. Dessa forma, teremos um modelo que se ajusta corretamente aos dados. Sendo assim, para este modelo o ideal seria uma profundidade de 3, pois ele chega a um Score de aproximadamente 75%.

Resposta:

A tcnica de validao-cruzada k-fold consiste nos seguintes passos:

  • 1- Dividir o conjunto total de dados em um nmero 'k' de subconjuntos;
  • 2- Um dos subconjuntos usado para teste, enquanto os outros subconjuntos restantes (k - 1) so utilizados para treinamento;
  • 3- Esse processo se repete 'k' vezes at que todos os conjuntos tenham passado por este processo, e dessa forma, seja possvel calcular a preciso do modelo;
  • 4- Em seguida, calculamos a mdia dos resultados para obter um resultado final.

Ou seja, durante todo esse processamento estamos utilizando o conjunto inteiro de dados, sendo dividido em 'k' vezes e realizando validao e teste. Dessa forma, utilizamos cada poro dos dados, ao invs de um nico subconjunto especfico, sem deixar de lado qualquer dado til para o treinamento.

Apesar do custo de processamento computacional ser caro, acaba por ser vivel, pois em projetos onde nos deparamos com uma quantidade de dados pequena podemos validar e testar de forma precisa com a quantidade limitada de dados. Entretanto, recomendvel que o nmero 'k' de divises no seja alto, uma vez que sempre voltamos a estaca zero quando testamos/validamos um novo subconjunto (k - 1), o que torna o custo computacional caro. Alm do mais, um nmero alto de divises tambm pode acabar por diminuir a preciso.

A busca em matriz (GridSearch) faz uso dos conjuntos de dados de validao para comparar qual o melhor modelo, por exemplo, o F1 Score que calculado no conjunto de validao cruzada em cada um dos modelos, sem ele no h como identificar qual modelo possui um melhor ajuste aos dados.

from sklearn.metrics import make_scorer
from sklearn.tree import DecisionTreeRegressor
from sklearn.model_selection import GridSearchCV

def fit_model(X, y):
    """ Desempenhar busca em matriz sobre o parmetro the 'max_depth' para uma 
        rvore de deciso de regresso treinada nos dados de entrada [X, y]. """
    
    # Gerar conjuntos de validao-cruzada para o treinamento de dados 
    cv_sets = ShuffleSplit(n_splits = 10, test_size = 0.20, random_state = 0)

    # TODO: Gerar uma rvore de deciso de regresso de objeto
    regressor = DecisionTreeRegressor()

    # TODO: Gerar um dicionrio para o parmetro 'max_depth' com um alcance de 1 a 10
    params = {'max_depth': range(1, 11)}

    # TODO: Transformar 'performance_metric' em uma funo de pontuao utilizando 'make_scorer' 
    scoring_fnc = make_scorer(performance_metric)

    # TODO: Gerar o objeto de busca em matriz
    grid = GridSearchCV(estimator=regressor, param_grid=params, scoring=scoring_fnc, cv=cv_sets)

    # Ajustar o objeto de busca em matriz com os dados para calcular o modelo timo
    grid = grid.fit(X, y)

    # Devolver o modelo timo depois de realizar o ajuste dos dados
    return grid.best_estimator_
